## 明确调研方向

在调研之前，首先要明确调研的目标和路线图，并将这些计划和预期与相关同事进行沟通。这一过程中，理清思路是关键。**与同事讨论是理清思路的好方法**。

调研的实施者与出题者通常不是同一个人，因此**要确认出题者的需求**。在开始调研之前，可以尝试**用自己的理解简要概述调研内容**，并与出题者确认。不明确需求可能会导致调研方向偏离，比如某次调研 GitHub Action 时，因为未能明确需求，结果关注点放在了错误的方向上。

## 把握调研重点

调研的关键在于「**判断获取的信息哪些对我们有价值**」，重在收缩范围。比如调研一本书，不应列出所有知识点，而要提取对我们有价值的部分。否则，这种流水账式的调研毫无意义。

在创业型公司，我们经常做自己不擅长或是新领域的事情，因此难以让新同事调研，因为他们可能无法判断哪些信息对我们的工作有价值。这就是为什么不建议新手直接看技术书籍，不是不看，而是不能漫无目的地看。

## 调研的是什么

搞清楚该技术解决了什么问题。我们调研的技术一般都是为了解决实际问题，因此要明确其解决问题的边界。

要比较各种解决方案，总有取舍，了解为何选择了某个方案而不是另一个。

确定该技术的使用场景，使用场景能帮助判断技术是否适合我们。

## 构造经典案例

调研中涉及许多抽象概念，例如 GitHub Action 中的 workflow、step、job 等，可能用文字难以说清。这时，构造经典的 demo 并将概念融入其中会有良好效果。demo 需要精炼，包含更多特性。

## 其他 ( 关于科普与调研之间的关系)

科普跟调研都有一个比较相似的点，那就是「 **用别人能听懂的话来说明一个别人没有接触过的知识** 」。此处我想起《科学有故事》栏目的作者汪洁，他就提到了 「科普作家的工作，是从信息的海洋中提出真实、准确、客观的信息，经过自己语言加工后再向大家普及， 让大家能更容易理解和吸收」。又或者说像是阮一峰的文章之所以那么受欢迎，因为他的文章其实跟科普文的的性质是很类似的。

总结来说，技术调研需要明确方向、把握重点、说明技术解决的问题、构建经典案例，并善于用易懂的语言传达复杂的概念。希望这些个人经验能对你的调研工作有所帮助。
